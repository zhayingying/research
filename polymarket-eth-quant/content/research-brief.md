# 调研任务书 · Polymarket / ETH / Quant 交易认知框架

作者: ziye  
创建日期: 2026-06-24

## 一、调研目标

建立一份面向主动交易者的机制型报告，回答四个核心问题：

1. 顶级交易员真正遵守的“铁律”是什么，为什么这些规则比单次预测更重要。
2. ETH 价格和 ETH 相关短周期市场会受哪些因素波动，哪些因素是短线、哪些是中长线。
3. 如果尝试用纯数学模型拟合市场并做预测，需要掌握哪些数学领域，以及它们分别解决什么问题。
4. 顶级量化公司如何把预测模型做成可交易系统，而不是停留在曲线拟合或 backtest 漂亮。

本报告服务的是一个准备研究 Polymarket / ETH 交易、同时想理解量化模型底层逻辑的读者。目标不是提供买卖建议，而是建立“可证据化、可复盘、可验证、可迭代”的交易认知框架。

## 二、交付要求

| 项目 | 要求 |
| --- | --- |
| 形式 | 先交付结构化 Markdown / JSON 资料，再生成可视化 HTML 报告 |
| 密度 | 表格、流程图和矩阵优先，少写空泛交易鸡汤 |
| 证据 | Polymarket / ETH 当前事实使用官方文档、监管文件、ETF/发行人资料、Ethereum 官方文档和可复核论文 |
| 口吻 | 解释机制，不提供具体交易指令 |
| 风险 | 明确区分模型可预测、可部分建模、不可稳定预测的区域 |
| 验证 | 来源链接检查、HTML 渲染检查、关键结论映射到 source id |

## 三、范围

包含：

- Polymarket 国际站与 Polymarket US 的基础市场结构、CLOB、API、费用、地域限制和监管风险。
- ETH 作为现货资产、ETF 标的、质押资产、gas/MEV/链上活动承载物和衍生品标的的波动来源。
- 顶级交易员的通用铁律：风险预算、仓位、赔率、流动性、执行、纪律、复盘、信息优势。
- 量化预测模型的数学基础和工程流程。
- 顶级量化公司的公开可观察方法论：科学研究流程、市场微观结构、alpha 研究、回测和风控。

排除：

- 不给具体买卖点、杠杆倍数、收益承诺。
- 不做“保证盈利策略”。
- 不复制闭源量化公司的商业秘密或不可验证传闻。
- 不把社交媒体热帖当事实源。

## 五、证据标准

| 等级 | 来源 | 用途 |

## 七、Reflect Loop 初始缺口

| Gap | 当前处理 |
| --- | --- |
| Polymarket US / 国际站监管状态变化快 | 优先官方 Polymarket US、CFTC、Polymarket docs；新闻只做 context |
| ETH ETF flows 每日变化 | 只写机制和数据源，不把单日 flows 当长期结论 |
| “顶级量化公司怎么做”闭源不可验证 | 使用公开论文、官方招聘/研究页面、市场微观结构文献抽象流程，不编造内部细节 |
| 纯数学拟合容易误导为“能预测一切” | 报告中必须设一章说明不可建模区域、过拟合和 regime shift |

## 八、一句话初步结论

顶级交易不是“预测最准”，而是把有限可预测性转化为可控风险下的正期望系统；ETH 和 Polymarket 的短周期市场尤其如此，模型只能处理概率、流动性、成本和状态变化，不能替代风险纪律、信息审计和执行约束。
